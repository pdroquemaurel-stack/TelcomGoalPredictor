# Pan-African Football Predictor MVP

Next.js + TypeScript + Prisma + PostgreSQL MVP for football predictions (player app + admin app), prepared for Render deployment.

## Quick stack
- Next.js 14 (App Router)
- Prisma ORM + PostgreSQL
- NextAuth (credentials)
- Tailwind CSS

---

## 1) Required environment variables

Copy:
```bash
cp .env.example .env
```

Set values:
```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/DB_NAME?schema=public"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="replace-with-a-long-random-string"
FOOTBALL_DATA_API_KEY=""
```

Notes:
- `DATABASE_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET` are required.
- `FOOTBALL_DATA_API_KEY` is optional for first deploy (needed only for real fixture sync).

## 3) Render deployment (recommended)

This repo contains `render.yaml` for Blueprint deployment.

### Step A — Deploy with Blueprint
1. Push this repository to GitHub.
2. In Render: **New** → **Blueprint**.
3. Select your repo.
4. Render creates:
   - Web service: `pan-african-football-predictor`
   - PostgreSQL: `pan-african-football-db`

### Step B — Confirm Render commands
From `render.yaml`:
- **Build Command**: `npm install && npm run prisma:generate && npm run build`
- **Pre-Deploy Command**: `npm run prisma:deploy`
- **Start Command**: `npm run start`

This ensures:
- Prisma client is generated during build.
- Migrations are applied before each deploy release.
- App start remains `next start`.

### Step C — Set environment variables in Render
In web service Environment:
- `DATABASE_URL` → linked from Render Postgres (auto from Blueprint)
- `NEXTAUTH_SECRET` → generated by Render (auto from Blueprint)
- `NEXTAUTH_URL` → set to your real app URL, e.g. `https://your-service.onrender.com`
- `FOOTBALL_DATA_API_KEY` → optional at first deploy; add later for fixture sync

### Step D — First seed (manual, safest)
After first successful deploy, open Render Shell in the web service and run:
```bash
npm run prisma:seed
```

Why manual: avoids accidental reseeding on every deploy while still allowing a reliable first initialization.

### Step E — Optional fixture sync
After adding `FOOTBALL_DATA_API_KEY`, log in as admin and use:
- `/admin/fixtures` → **Sync fixtures from API**

## 5) Troubleshooting

### Build fails with missing tables during `next build`
DB-backed routes are configured as dynamic, so build should not query Prisma tables. Ensure you are using current code and rerun:
```bash
npm run build
```

### Auth issues after deploy
- Verify `NEXTAUTH_URL` exactly matches public Render URL.
- Verify `NEXTAUTH_SECRET` is set.

### DB errors on deploy
- Confirm `DATABASE_URL` is linked to Render Postgres.
- Confirm pre-deploy migration command ran (`npm run prisma:deploy`) in deploy logs.
